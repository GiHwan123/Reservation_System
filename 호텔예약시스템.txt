호텔 예약 시스템

예약 - num,시간,회원id
회원 - num,id,password,reservation

-회원가입
1. form에 입력받아 입력받은 id와 password를 Customer 테이블에 저장.
2. 저장 후 성공 main 페이지로 넘어옴

회원가입 구현 중 어려움을 겪은 점
1. 넘겨주는 command에 오타가 난 걸 20분동안 찾지 못해 헤맸음..


-로그인

1. Session을 활용해 입력받은 id 값과 password 값이 Customer 테이블 내에
존재하는지, 일치하는지 여부 확인.
2. 존재/비존재 일치/불일치 여부에 따라 jsp로 처리 후 세션값 넘겨줌
3. 존재/일치 -> login.jsp 파일로 이동
4. 비존재or 불일치 -> 안내메세지와 함께 재로그인 요청(login.jsp 파일로 이동)

로그인 구현 중 어려움을 겪은 점
1. 임시로 person/11 id,pw로 로그인 후 세션을 유지한 채 main.html로 넘어가기
	-> main.html로 넘어가는것이 아닌 loginmain.jsp로 넘어가서
	우측상단	login,register 버튼을 logout 버튼으로 변경
2. alert 창 내부에 입력된 session 값을 띄워주며 로그인 성공을 알리고 싶음
	->response.Write로 해결

예약 추가 구현 중 어려움을 겪은 점
1. 예약테이블이 생성되지않고있음.
	-> 원인 : Date라는 변수명을 사용하였는데
	오라클 DB에서는 기본변수명처럼 취급하기때문에
	사용이 불가능.
	-> 해결 : resDate라는 변수명으로 변경하였더니 해결.
2. 입력받은 날짜 값으로 reservation 테이블 내에서 해당 날짜가 저장된
예약이 존재하는지 확인하려고 하지만, JPA를 활용해 기본키가 아닌 날짜값을
탐색하는 방법을 모르겠음.
	-> JDBC와 JPA를 둘 다 활용해 JDBC를 이용해 특정 주문을 탐색,
	JPA를 활용해 일대다/다대일 관계의 엔티티인 주문 데이터를 삽입.
